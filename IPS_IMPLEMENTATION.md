# International Patient Summary (IPS) Implementation

Symphony now implements the **IPS (International Patient Summary)** standard as defined by the HL7 FHIR IPS Implementation Guide.

## Overview

The IPS is a minimal and non-exhaustive patient summary designed for unplanned, cross-border care. Our implementation creates FHIR Composition resources that comply with the IPS structure and include narrative sections for human readability.

## IPS Implementation Guide Reference

Based on: [HL7 FHIR IPS Implementation Guide](https://build.fhir.org/ig/HL7/fhir-ips/StructureDefinition-Composition-uv-ips.html)

## Key Features

### 1. IPS-Compliant Composition Structure

Symphony creates Composition resources with the following IPS-specific characteristics:

- **Resource Type**: `Composition`
- **Type Code**: LOINC `60591-5` (Patient Summary Document)
- **Category**: AI Generated (`http://terminology.hl7.org/CodeSystem/v3-ActCode`)
- **Status**: `final`
- **Title**: "International Patient Summary - AI Generated"

### 2. Required IPS Sections

#### Active Problems Section
- **LOINC Code**: `11450-4` (Problem list - Reported)
- **Content**: Current active medical conditions
- **Narrative**: HTML table with problem names, onset dates, and codes

#### Medication Summary Section
- **LOINC Code**: `10160-0` (History of Medication use Narrative)
- **Content**: Current medications and their status
- **Narrative**: HTML table with medication names, status, and codes

#### Allergies and Intolerances Section
- **LOINC Code**: `48765-2` (Allergies and adverse reactions Document)
- **Content**: Known allergies and intolerances
- **Narrative**: HTML table with allergens, onset dates, and codes

### 3. Optional IPS Sections

#### Vital Signs Section
- **LOINC Code**: `8716-3` (Vital signs)
- **Content**: Recent vital sign measurements
- **Narrative**: HTML table with vital signs, values, and dates

#### Diagnostic Results Section
- **LOINC Code**: `30954-2` (Relevant diagnostic tests/laboratory data Narrative)
- **Content**: Laboratory results and diagnostic findings
- **Narrative**: HTML table with test names, results, and dates

#### Procedures Section
- **LOINC Code**: `47519-4` (History of Procedures Document)
- **Content**: Medical procedures performed
- **Narrative**: HTML table with procedure names, dates, and codes

#### Care Team Section
- **LOINC Code**: `85847-2` (Care Team)
- **Content**: Healthcare encounters and care team information
- **Narrative**: HTML table with encounter types and descriptions

### 4. Extended Sections (Symphony-Specific)

#### Care Gaps Section
- **LOINC Code**: `18776-5` (Plan of care note)
- **Content**: AI-identified care gaps and recommendations
- **Narrative**: HTML list with care gap descriptions

#### Data Quality Section
- **LOINC Code**: `85899-3` (Data source)
- **Content**: Data quality assessment and limitations
- **Narrative**: HTML list with data quality notes

## Implementation Details

### Narrative Generation

Each section includes rich HTML narratives that are:
- **Human-readable**: Formatted tables and lists for clinical review
- **FHIR-compliant**: Valid XHTML with proper namespace declarations
- **Structured**: Consistent formatting across all sections
- **Safe**: HTML-escaped content to prevent injection attacks

### Empty Section Handling

Sections with no data include:
- Appropriate `emptyReason` codes (e.g., "nilknown")
- Meaningful narrative text explaining the absence of data
- Proper LOINC codes for section identification

### Claude AI Integration

The IPS Composition is populated with data generated by Claude AI, which:
- Analyzes longitudinal patient FHIR data
- Extracts relevant clinical information
- Generates structured summaries with proper medical terminology
- Identifies care gaps and data quality issues

## Usage

### Creating an IPS Composition

```typescript
import { IPSService } from './lib/ips-service'

const composition = IPSService.createIPSComposition(
  summaryData,
  'Patient/123',
  'Symphony AI Clinical Summary Generator'
)
```

### Viewing Generated IPS Documents

1. Complete the 5-step workflow in Symphony
2. Use the "Write to HAPI Server" function to materialize the IPS
3. View the created Composition resource via the provided links
4. Access the HAPI FHIR server to see the full IPS document

## FHIR Server Integration

### Hosted HAPI FHIR Server

Symphony connects to the public HAPI FHIR test server:
- **Base URL**: `https://hapi.fhir.org/baseR4`
- **Web Interface**: `https://hapi.fhir.org`
- **Access**: Public test server with sample data

### Resource Creation

When materializing summaries, Symphony creates:
1. **IPS Composition**: Main summary document with narratives
2. **DocumentReference**: Reference to the JSON summary data
3. **Provenance**: Tracking information for AI generation

### Viewing Resources

After materialization, direct links are provided to:
- View the created Composition resource
- Access the DocumentReference with embedded JSON
- Browse all Composition resources on the server
